# Social Science Explainer

## 定位任务

- 目标不是做摘要，而是把“研究者如何把一个模糊困惑变成可讨论、可论证、可反驳的研究问题”讲明白。
- 默认输出中文大众文章；除非用户要求学术书评、课堂报告、结构化笔记或其他语言。
- 同时交代八件事：作者提出了什么问题，这个问题在学术上为什么构成 puzzle，案例或对象的背景是什么，作者怎样研究，发现了什么，提出或修正了什么理论，作品的价值与局限在哪里。
- 文章最后最好让读者感到：社会科学不是结论堆砌，而是一种追问世界的方式。

## 先选叙述方式

### 故事化导读版

- 默认优先考虑这一模式，尤其适用于面向大众解释论文、专著或案例研究。
- 结构顺序优先采用：怪现象或关键场景 -> 为什么这让人困惑 -> 旧解释卡在哪里 -> 研究者如何继续追问 -> 作者如何研究 -> 发现了什么 -> 这改变了什么理解。
- 叙事对象不是作者生平，而是研究推进过程本身。
- 可以适度制造悬念，但不要为了讲故事而歪曲论证顺序或证据强度。

### 结构化解读版

- 适用于用户明确要提纲、笔记、课程汇报或学术化分节输出。
- 保持清楚分节，但仍然建议在开头先用 1-2 段建立问题意识。

## 适用范围

### 适合使用本 skill 的任务

- 解读社会学、政治学、人类学、经济学、历史学、传播学、教育学、法学、地理学等社会科学研究作品。
- 将学术专著、期刊论文、章节、研究报告改写成面向大众的可读文章。
- 需要解释研究问题、方法、证据、理论贡献、局限，而不只是讲“这本书说了什么”。
- 需要把抽象概念、方法术语和学术意义翻译成普通读者能理解的语言。

### 不要硬套本 skill 的任务

- 纯新闻评论、时事快评、观点社论。
- 自助励志书、商业畅销书、小说、散文或非研究型随笔。
- 自然科学、工程技术、医学论文的专业解读。
- 用户只要极短摘要时；此时只保留问题、发现、意义三件事即可，不必铺开完整框架。

## 先判断材料状态

### 材料完整

- 用户提供全文、PDF、长摘录、完整笔记或足够多的关键章节。
- 可以执行完整工作流。

### 只有标题、书名、摘要或 DOI

- 先确认元数据，再根据可得材料做“侦察版解读”。
- 明确区分哪些判断来自题目、摘要、导论、评论或公开简介，哪些仍然不能确定。
- 如果环境允许联网，优先查作者摘要、出版社页面、期刊页面、书评或访谈；不要假装已经读过全文。

### 只有局部材料

- 只对可见范围内的论证做判断。
- 不把单章、单节或片段强行外推为整本书或整篇论文的最终结论。

## 先识别作品类型

### 研究论文

- 重点看：研究问题、学术背景、文献缺口、案例背景、研究设计、证据、发现、理论贡献。
- 需要细则时，读取 [references/paper-mode.md](references/paper-mode.md)。

### 研究专著

- 重点看：大问题、总体论点、论证推进方式、案例布局、长期解释力。
- 需要细则时，读取 [references/book-mode.md](references/book-mode.md)。

### 章节、研究报告或混合材料

- 先判断它更接近论文还是专著，再借用对应模式。
- 如果属于政策报告、智库报告或综述性作品，重点放在问题设定、证据来源、解释框架和适用边界。

## 核心工作流

### 1. 锁定研究谜题

- 先用 1-2 句话回答：这项研究到底想解释什么现象、矛盾、反常识结果或历史变化。
- 说明这个问题为什么值得问：它挑战了什么常识、填补了什么空白、回应了什么现实困惑。
- 如果作者的真正问题被术语包住了，要先拆开术语，再还原成读者能感到“这确实是个问题”的语言。

### 2. 补上学术背景

- 交代作者是在和哪些已有解释对话：主流理论怎么解释这个问题，它们哪里解释得不够。
- 不要把“文献综述”整段搬给读者；只保留 2-4 个最关键的既有思路或争论。
- 要明确说出 puzzle 在哪里：是现实结果反常、既有理论冲突，还是老问题在新材料下解释不通。

### 3. 交代案例或对象背景

- 如果研究依托单一案例、关键事件、特定城市、组织、社群或国家，先交代最小必要背景。
- 背景的目标不是补百科，而是让读者知道：这个案例发生在什么情境里，为什么它能承载作者的问题。
- 只保留与研究问题直接相关的背景信息，例如制度环境、历史节点、关键行动者、时间线起点。

### 4. 重建研究路径

- 用普通话解释作者是如何研究这个问题的：访谈、田野、档案、案例比较、历史材料、统计数据、实验、文本分析、形式模型，或它们的组合。
- 方法不是单独列术语，而是回答“作者靠什么材料、什么比较、什么观察来支撑自己的说法”。
- 进一步讲清研究设计的论证逻辑：作者拿什么现象来区分竞争解释，为什么这些证据如果出现 A 就更支持理论 A、如果出现 B 就更支持理论 B。
- 如果作者是在“排除别的解释”，要把排除过程写出来，而不是只说“作者证明了”。
- 如果作品是理论型或解释型而非严格实证型，也要讲清它的推理路径，而不是强行套经验研究模板。

### 5. 提炼核心发现

- 只保留 2-4 个最重要的发现、机制或论证动作。
- 每个发现都回答三个问题：作者发现了什么，这个发现纠正了什么旧看法，它是靠什么证据或推理站住的。
- 避免把章节目录改写成条目；要按解释逻辑而不是原始顺序组织。

### 6. 说明理论与学术意义

- 明确这项研究是提出新概念、新机制、新分类，还是修正旧理论、重新解释旧材料、扩大既有理论适用范围。
- 学术意义不等于“很重要”。要具体写清：它改变了学界原本怎么看这个问题。
- 有些作品的价值主要是经验发现，有些主要是理论重构，有些主要是方法创新；不要机械地三项都写满。

### 7. 补上局限与边界

- 至少指出 1-2 个真正有分量的局限：样本边界、时代背景、因果识别、概念模糊、外推风险、替代解释、材料偏差。
- 局限不是礼貌性结尾，而是帮助读者理解“这项研究能解释什么，不能解释什么”。
- 材料不足时，直接说明判断边界，不要编造批判。

### 8. 保留探索的乐趣

- 至少写一小段说明这项研究最迷人的地方：作者如何从一个看似平常的现象里挖出深层结构，或如何用巧妙证据把抽象理论落到现实。
- 这一段不是抒情，而是让读者看到研究中的发现感、辨认力或解释力。
- 如果作品本身很枯燥，也要尽量指出它在哪个问题上完成了真正的“看见”。

### 9. 面向大众重写

- 优先使用“日常现象或直觉困惑 -> 研究问题 -> 学术背景/puzzle -> 案例背景 -> 研究路径 -> 关键发现 -> 学术意义 -> 局限 -> 探索的乐趣”的结构。
- 如果采用故事化导读版，则进一步改写为“场景开头 -> puzzle 浮现 -> 旧理论受挫 -> 研究者提出新解释 -> 用案例/材料层层推进 -> 得出结论与余味”。
- 在作者或作品第一次出场时，先完成“对象引介”：至少交代作品名、作者姓名或身份、它要解释什么问题；在此之前不要直接写“这篇论文”“作者认为”。
- 专业概念第一次出现时，先给通俗定义，再给一个短例子。
- 明确区分三层内容：作者主张、作者证据、解释者延伸判断。
- 避免写成学术腔摘要，也避免写成脱离原作的泛泛现实评论。

## 默认输出骨架

```markdown
# [作品名]：研究者到底想弄明白什么

## 一、这项研究从什么问题出发
[交代研究对象、谜题、现实意义或反常识之处]

## 二、为什么这在学术上是个 puzzling 的问题
[交代作者在回应哪些既有解释，这些解释哪里不够]

## 三、案例或研究对象的背景是什么
[如果是案例研究，在这里交代读者理解后文所需的最小背景]

## 四、作者是怎么研究这个问题的
[用非术语化语言解释材料、方法、比较路径或推理路径]

## 五、这项研究发现了什么
### 1. [关键发现或机制]
[解释它回答了什么问题]
### 2. [关键发现或机制]
[解释它推翻或修正了什么旧看法]
### 3. [可选]
[按需要继续]

## 六、它提出了什么解释或理论
[说明新的概念、机制、框架或理论修正]

## 七、这项研究有什么学术意义
[说明它改变了学界或读者原本怎样理解这个问题]

## 八、这项研究的局限在哪里
[指出边界、盲点、替代解释或外推限制]

## 九、为什么这项探索值得读
[让读者感到研究中的洞察力、方法感和探索乐趣]
```

## 故事化导读版骨架

```markdown
# [作品名]：一个看起来解释不通的现象

[用一个场景、反常识结果或关键时刻开场]

## 一、奇怪的地方在哪里
[说明现象为何不只是“重要”，而是“难解释”]

## 二、旧解释为什么不够
[用 2-4 个主要理论或常见说法展示它们各自卡住的地方]

## 三、研究者换了一个什么角度
[交代作者如何重新定义问题，提出新的解释路径]

## 四、作者怎样一步步检验这个想法
[讲清案例背景、材料、方法与论证推进]

## 五、研究最后看见了什么
[提炼 2-4 个核心发现]

## 六、这改变了我们如何理解这个世界
[说明理论意义、现实启发与边界]
```

## 写作约束

- 不要按章节顺序平铺复述，除非用户明确要求章节导读。
- 不要只抄摘要、目录或结论句。
- 不要把研究问题写成孤零零的一句结论；要交代它相对于既有解释为什么成立为一个 puzzle。
- 如果作品依托具体案例，不要直接闯入论证；先补足读者理解案例所需的最小背景。
- 不要把“讲故事”理解成煽情写作；故事感来自研究推进过程中的问题、障碍、转折与发现。
- 不要让作者或作品“悬空出场”；首次出现时必须让读者知道“这是哪篇作品、谁写的、它在解释什么”。
- 不要把方法写成黑箱术语，例如只说“用了回归分析”“做了民族志”而不解释这些方法在这里试图解决什么问题。
- 不要把研究设计写成工具清单；要写成“作者如何让自己的解释比别的解释更难被推翻”。
- 不要把“学术意义”写成空话，例如“拓展了研究视野”“具有重要参考价值”。
- 当材料有限时，明确写出“以下判断基于摘要/导论/公开简介/可见片段”。
- 如果用户要更学术、更结构化的版本，可以提升概念密度，但仍然要保留可读性。
