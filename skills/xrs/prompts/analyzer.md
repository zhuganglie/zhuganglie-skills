## 任务

### 1. 角色识别

只保留满足以下任一条件的角色：

- 出场至少 2 次
- 是情节关键角色

为每个角色生成可直接用于图像生成的外观描述。

格式建议：

`"{年龄段}{性别}，{服饰}，{体貌特征}，{独特标志或气质}"`

要求：

- 原文没有外貌时，可根据身份、年代和场景做保守推断
- 描述尽量具体，但不要加入原文没有支持的强细节
- 后续所有帧都会复用该描述，所以要稳定、完整

### 2. 结构标注

识别以下信息：

- 直接引语和说话人
- 时间变化
- 地点变化
- 人物进出场
- 情节关键点

`plot_marker` 只允许以下 6 种值：

- `开场`
- `冲突引入`
- `发展`
- `转折`
- `高潮`
- `结局`

### 3. 分帧规划

按以下优先级规划画面：

| 优先级 | 条件 | 动作 |
|---|---|---|
| 1 | 开场 / 冲突引入 / 转折 / 高潮 / 结局 | 必须独立成帧 |
| 2 | 场景切换且当前内容已较饱满 | 新建一帧 |
| 3 | 连续对话较多 | 单独给对话帧 |
| 4 | 单帧承载信息过多 | 在段落或对话间隙强制切开 |
| 5 | 过渡段较短 | 与前后帧合并 |

镜头建议：

- 开场 / 结尾：远景或全景
- 对话密集：中景或过肩
- 情绪高潮 / 关键细节：特写
- 动作场景：全景或中景

## 输出格式

只输出 JSON，不要解释，不要 Markdown 之外的文字：

```json
{
  "metadata": {
    "title": "标题",
    "author": "作者或佚名",
    "word_count": 2800
  },
  "characters": {
    "角色名1": "完整外观描述",
    "角色名2": "完整外观描述"
  },
  "frame_plan": [
    {
      "frame_number": 1,
      "plot_marker": "开场",
      "text_start": "原文实际存在的前 15-20 字",
      "text_end": "原文实际存在的后 15-20 字",
      "scene": {
        "time": "白天",
        "location": "具体地点",
        "atmosphere": "环境氛围"
      },
      "characters_in_frame": ["角色名1"],
      "has_dialogue": false,
      "suggested_shot": "全景"
    }
  ]
}
```

### 硬性要求

1. `text_start` 和 `text_end` 必须都来自原文。
2. `frame_plan` 必须按顺序覆盖完整文本，避免明显遗漏。
3. `characters_in_frame` 只列真实出场角色。
4. `suggested_shot` 要有多样性，不要所有帧都同一种镜头。
5. `plot_marker` 必须只使用上面列出的 6 个值。
