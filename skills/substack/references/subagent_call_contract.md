# 子代理调用包模板

主 skill 每次调用子代理，都应提供一个最小且完整的调用包。缺少任一字段时，不应继续下游执行。

## 必填字段

- `阶段`：planning / writing / review / publishing
- `agent`：本轮执行的唯一 agent 名称
- `任务目录`：当前任务工作目录的相对或绝对路径
- `输入文件`：本轮必须读取的文件列表
- `输出文件`：本轮必须生成的文件列表
- `本轮目标`：一句话说明这一步要解决什么问题
- `边界`：这一步不能做什么，例如“不得向用户提问”“不得改写整篇”“不得引入新事实”
- `证据规则`：是否保留来源键、是否允许 `[待核实]`、是否必须标注核查日期
- `返回格式`：要求使用固定汇报格式

## 推荐调用包

```text
阶段: writing
agent: ContentWriter
任务目录: runs/2026-03-13-example-topic
输入文件:
- argument_structure.md
- detailed_outline.md
- annotated_bibliography.md
输出文件:
- sections/03-case-study.md
本轮目标:
- 完成第 03 章初稿，聚焦核心案例与机制解释
边界:
- 只写本章
- 不得覆写 draft_v1.md
- 不得向用户提问
证据规则:
- 保留来源键
- 证据不足时使用 [待核实]
返回格式:
- 状态 / 已生成文件 / 摘要 / 待确认问题 / 默认假设 / 风险或阻塞
```

## 协调器检查清单

发起调用前确认：

1. 所有输入文件存在
2. 输出路径位于任务目录内
3. 输出文件名与阶段约定一致
4. 当前步骤和下一步已更新到 `workflow_state.md`
5. 若为审核点，用户反馈边界已写入 `revision_decision.md` 或 `workflow_state.md`
