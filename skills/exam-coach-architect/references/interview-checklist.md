# 访谈清单

只在信息缺失或歧义时读取本文件。目标是尽快拿到足以生成工作区的最小输入，而不是把用户拖进长问卷。

## 必问项

1. 科目与考试体系
2. 输出目录或工作区根目录
3. 工作区文档语言
4. 教练与学生的交互语言
5. 主要考核模块 / 题型 / paper
6. 教练人格设定

## 推荐补充项

- 目标分数 / 目标等级
- 当前水平或最近一次模考表现
- 是否已有 syllabus、rubric、past papers、sample responses
- 默认训练节奏，例如每周几次、每次多久

## 提问策略

- 一次只问缺失的关键项，不要把已知信息再问一遍。
- 如果用户已经给了考试名称和目标目录，下一轮优先补语言、模块、persona。
- 如果模块缺失但考试结构非常标准，可以给出一个暂定列表并请用户确认，而不是完全停住。
- 如果当前目录明显是 skill 目录、工具目录或系统目录，优先确认输出路径。

## 默认假设规则

- 没有目标分数时，保留空字段，不要编造。
- 没有当前水平时，在 `LEARNER.md` 写成“待诊断”。
- 没有 syllabus / rubric / examples 时，只创建骨架文件，不要虚构内容。
- 用户没有说明教练风格时，可以先采用“严格但不替做、以诊断为先、微步推进”的默认人格，并明确告知这是临时默认值。
