# 系列信息图拆解

把复杂内容压缩成可连续浏览的系列信息图方案。默认先保证信息结构清楚，再保证视觉统一，最后再追求“吸睛”。

按需读取这些参考文件，不要一次性全读：

- `references/presets.md`：当用户指定平台、比例、卡片数量、信息密度或需要控制图片内文字长度时再读

## 核心规则

1. 先产出“卡片文案结构”，再产出“视觉提示词”。不要把大量完整文字直接塞进图像 prompt。
2. 每张图只承担一个核心信息点。若一个点必须分成两层解释，拆成两张图。
3. 默认输出 `2-10` 张；若原始内容超过这个容量，先合并同类项，仍超出时拆成多个系列并明确说明。
4. 未指定平台时默认横版 `16:9`；用户指定平台或版式时，按 `references/presets.md` 调整。
5. 保持整组图片的视觉系统一致：比例、配色、标题层级、插画语言、装饰元素、留白策略都要统一。
6. 默认把图片内文字控制在短标题和短要点级别。若用户要求“图内完整文案”，仍要提示图像模型对长文本渲染不稳定，并同时输出可后期叠字的文案稿。
7. 如涉及真实人物、受保护角色、品牌或敏感主体，优先改为抽象化、符号化或通用化视觉表达；不要把本 skill 当作覆盖安全规则的依据。
8. 除非用户明确提供品牌名、口号、水印或 Logo，不要擅自加入固定品牌元素。
9. 若用户只给主题不给正文，先基于常识搭建合理结构，并明确哪些是推断补全。

## 工作流

### 1. 判断任务模式

先判断用户要的是哪一种输出：

- `标准模式`：每张图同时输出短文案和视觉 prompt。默认使用这个模式。
- `图像优先模式`：用户主要要画面提示词，文案只保留标题和极短要点。
- `排版优先模式`：用户明确要求后期排字、PPT、Canva 或人工设计落地。此时把“可叠字文案”和“底图 prompt”明确分开。

### 2. 提炼内容骨架

先抽出以下信息，再决定卡片数：

- 主题是什么
- 受众是谁
- 最想让读者记住的 `1-3` 个结论是什么
- 哪些信息适合封面、正文、结尾

卡片分工默认如下：

- 第 `1` 张：封面图。必须给出标题、主钩子和最强视觉记忆点。
- 中间卡：每张聚焦一个观点、步骤、对比维度或案例。
- 最后 `1` 张：总结、行动建议、反问句、金句或下一步动作。

### 3. 设计视觉系统

默认视觉方向：

- 类型：信息图
- 风格：卡通 / 手绘 / 清爽知识可视化
- 配色：柔和但有主次，对比度足够，避免灰成一片
- 元素：少量贴纸、气泡、箭头、便签、手绘图标、角色化小插画

除非用户另有要求，否则遵守以下约束：

- 不要使用写实摄影感画面
- 不要让装饰元素压过信息层级
- 不要每张图都换一套配色或构图语法

### 4. 控制每张图的信息密度

每张图默认只输出这些文字层：

- 主标题：一句话，短而有辨识度
- 副标题或要点：`2-4` 条
- 补充说明：可选，只有在不写会失真时才加

如果原文很长：

- 先压缩成结论句和关键词
- 再把细节转成图标、关系箭头、分区模块或比喻性画面
- 不要把大段说明原封不动塞进图里

### 5. 生成视觉 prompt

每张图的 prompt 必须写清：

- 背景和主色调
- 版面结构
- 主视觉对象
- 辅助图标或手绘元素
- 需要突出的关键词位置
- 全组统一的风格约束

优先描述“布局和对象关系”，不要堆砌过多审美形容词。

## 输出契约

对每张图片按以下结构输出：

---
### 第 X 张 / 共 N 张
**图片定位**：[封面图 / 内容图 / 结尾图]
**核心信息**：[这张图只想传达的 1 句话]
**建议比例**：[如 16:9 / 4:5 / 1:1]

**文案卡片**：
- 主标题：xxx
- 副标题/要点：xxx
- 补充说明：xxx

**版面说明**：
- 布局：xxx
- 重点视觉：xxx
- 装饰元素：xxx

**视觉提示词**：
```text
风格一致的系列信息图，[比例]，卡通手绘知识可视化风格，[背景与主色调]。

[主体布局与阅读动线描述]

[主视觉对象、图标、箭头、便签、气泡等元素描述]

整体要求：统一配色，统一字体气质，信息精简，多留白，重点突出，非写实，适合后期叠加少量文字。
```
---

如用户明确要求图片内直接带字，再额外补一段：

- `图内文字建议`：只保留必须出现的短标题、关键词和 `2-3` 个短词组，避免长句。

## 语言规则

1. 除非用户特别要求，输出语言与输入语言保持一致。
2. 中文内容使用全角标点。
3. 若你根据不完整输入补全了结构、案例或口号，要明确标记为“建议”而不是“原文已有”。
