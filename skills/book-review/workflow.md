# 书评分析工作流程

## 1. 先判定任务模式

先识别用户给了什么材料，以及他真正要什么输出。

| 输入类型 | 默认处理方式 |
|---------|-------------|
| **PDF 文件路径 / 长文本** | 直接读取材料并进入分析 |
| **复制粘贴的章节或片段** | 进行局部分析，并在开头说明覆盖范围受限 |
| **书名 / DOI / 作者 + 题目关键词** | 先确认元数据，再检索出版社页面、学术书评与相关文献 |
| **只要求某个模块** | 只执行相关模块，避免完整铺开 |

再识别输出模式：

- **单次整合输出**：默认模式。用户没有要求分步推进时，直接完成完整分析。
- **分步交互分析**：仅在用户明确要求逐步推进，或材料极长且目标不清时使用。
- **快速侦察版**：用户只要判断一本书是否值得读、贡献何在、局限何在时，压缩为核心论点 + 文献定位 + 关键批判。

## 2. 收集最小必要信息

只补问会实质改变分析的关键信息；能从材料或网络直接确认的，不必反复询问。

优先确认：

- 书籍完整标题与作者
- 出版年份与出版社
- 用户的用途：课程作业、研究准备、书评写作、口头报告
- 是否只关注特定模块

如果用户只给书名，先自行检索并补全这些信息，再进入分析。

## 3. 决定是否创建工作空间

不要默认创建 `notes/`。仅在以下情形创建：

- 任务会跨多轮推进
- 材料很长，需要保存中间产物
- 用户明确要可复用笔记或最终稿文件

若需要创建，可使用如下结构：

```text
notes/
├── 00-metadata.md
├── 01-theory.md
├── 02-method.md
├── 03-literature.md
├── 04-critique.md
├── 05-future.md
└── final-review.md
```

若不需要创建工作空间，直接在当前上下文中完成分析。

## 4. 先做初步提取

在正式展开前，先压缩出四项判断：

1. 一句话核心论点
2. 研究类型：实证、规范、形式，或其混合
3. 主要证据类型：案例、档案、统计、文本诠释、形式模型
4. 子领域定位：比较政治、国际关系、政治理论、公共行政

这一步决定后续模块的权重分配。

## 5. 按模块展开，但按需加权

推荐顺序仍然是：

```text
模块一（理论）→ 模块二（方法）→ 模块三（文献）→ 模块四（批判）→ 模块五（未来）
```

但不要机械平均用力。按子领域调整重心：

- **比较政治 / 公共行政的实证著作**：重点放在理论机制、研究设计、替代解释。
- **国际关系著作**：重点放在理论传统定位、层次分析、时期与结构条件。
- **政治理论著作**：重点放在概念界定、规范前提、推理链条与竞争理论回应。
- **形式模型著作**：重点放在假设、均衡、比较静态、现实对应关系。

模块间依赖：

- 模块四依赖模块一和模块二的结果。
- 模块三可并行推进，但最终必须服务于模块四，而不是单独堆砌文献。

## 6. 构建学术对话网络

优先使用 web 搜索与页面打开能力，而不是泛泛浏览。

### 来源优先级

按以下顺序找材料：

1. 出版社页面、作者页面、目录、导论摘要
2. 学术期刊书评、review essay、symposium
3. 领域综述、handbook 章节、Annual Review 类文章
4. 引用该书或与其争论的后续论文
5. 一般网页内容，仅作为补充背景

### 检索目标

- 识别作者显性引用的传统和核心前辈
- 找出主要支持者、批评者和替代路径
- 判断该书是提出新机制、修正旧理论，还是重组既有辩论

### 可复用检索式

```text
"[书名]" review
"[作者名]" "[书名关键词]"
"[书名关键词]" "political science" review
"[子领域]" review essay handbook
"[核心概念]" debate political science
```

### 并行检索任务

必要时使用 explorer / worker 子代理，给出明确边界，例如：

- 检索作者其他相关著作
- 检索主要批评性书评
- 检索近五到十年引用并修正该书的论文

对“隐含对话”的判断必须有文本证据或稳定的领域辩论作支撑；如果是推断，明确标注为推断。

## 7. 整合输出

默认整合为五模块结构；若用户只要聚焦分析，则保留相关模块并补一个简短结论。

整合时执行以下约束：

- 不复述目录或章节推进
- 每个主要批判后紧跟影响和替代方案
- 在材料有限时，把范围限制写在开头
- 在标题级结构中体现理论、方法、文献、批判、延伸，而不是“第一章、第二章”

## 8. 最终检查

- [ ] 是否先界定了材料范围与输出模式
- [ ] 是否把理论模型分解清楚
- [ ] 是否做了与研究类型相匹配的方法评估
- [ ] 是否构建了主动的文献对话，而非被动摘要
- [ ] 是否把高杠杆批判和替代方案对应起来
- [ ] 是否避免了空泛的“未来可进一步研究”

## 9. 错误处理

| 情况 | 处理方式 |
|------|---------|
| 材料不完整 | 明确覆盖范围限制，并说明哪些结论只能暂时成立 |
| 只有书名，没有正文 | 以元数据、书评、综述和引用链构建侦察版分析 |
| 非政治科学著作 | 说明超出最佳适用范围，保留可迁移框架并注明局限 |
| 用户要求简短 | 压缩篇幅，但保留核心论点、关键批判与文献定位 |
| 学术检索受限 | 说明哪些文献判断依赖现有可见材料，避免过度断言 |
