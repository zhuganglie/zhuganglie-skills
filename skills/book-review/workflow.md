# 书评分析工作流程

## 概述

本workflow指导如何通过多轮对话完成一本学术专著的深度批判性分析。

---

## 阶段一：输入确认与准备

### 确认输入来源

询问用户提供的材料类型：

| 输入类型 | 处理方式 |
|---------|---------|
| **PDF文件路径** | 使用 `read` 工具读取内容 |
| **复制粘贴的文本** | 直接分析用户提供的内容 |
| **书名/DOI** | 使用 `webfetch` 检索书籍信息和书评 |
| **章节片段** | 分析片段，但注明覆盖范围有限 |

### 初步信息收集

需要确认：
- 书籍完整标题和作者
- 出版年份和出版社
- 用户分析目的（课程作业？研究准备？书评写作？）
- 是否有特别关注的模块

---

## 阶段二：创建分析工作空间

### 使用 planning-with-files skill

```
创建 notes/ 目录存放中间产物：
notes/
├── 00-metadata.md      # 书籍基本信息
├── 01-theory.md        # 理论框架笔记
├── 02-method.md        # 方法论笔记
├── 03-literature.md    # 文献对话笔记
├── 04-critique.md      # 批判要点
├── 05-future.md        # 未来方向
└── final-review.md     # 最终报告
```

### 初步提取

首轮对话完成：
1. 识别核心论点（一句话）
2. 识别研究类型（小N/大N/混合）
3. 识别主要案例/数据来源
4. 记录到 `notes/00-metadata.md`

---

## 阶段三：分模块深度分析

### 分析顺序建议

```
模块一（理论）→ 模块二（方法）→ 模块三（文献）→ 模块四（批判）→ 模块五（未来）
```

### 每模块执行步骤

1. **加载模块指引**：读取对应的 `modules/*.md` 文件
2. **执行分析**：按指引完成分析任务
3. **保存中间产物**：写入对应的 `notes/*.md` 文件
4. **确认继续**：询问用户是否有补充问题

### 模块间依赖

- 模块四（批判）依赖模块一二的分析结果
- 模块三（文献）可与其他模块并行执行

---

## 阶段四：文献检索与学术对话构建

### 检索策略

使用 `webfetch` 工具执行：

```
# Google Scholar 检索
https://scholar.google.com/scholar?q="[作者名]"+OR+"[书名关键词]"

# 检索该领域的综述文章
https://scholar.google.com/scholar?q="[子领域]+review+OR+handbook"
```

### 可并行的检索任务

使用 `task` agent 并行执行：
- 检索作者其他著作
- 检索主要批评者的文章
- 检索近期引用该书的论文

---

## 阶段五：整合与输出

### 整合步骤

1. 读取所有 `notes/*.md` 中间产物
2. 按输出结构模板整合
3. 检查质量标准
4. 输出到 `notes/final-review.md`

### 最终检查清单

- [ ] 按分析维度组织（非章节顺序）
- [ ] 每模块包含评析/批判
- [ ] 主动构建了文献对话网络
- [ ] 每个批判配套替代方案
- [ ] 因果模型已分解为文字链条
- [ ] 充分展开，长篇深度分析

---

## 用户交互模式

### 单次完整分析
用户提供完整材料，一次性完成五模块分析

### 分步交互分析
每完成一个模块，确认后再继续下一模块

### 聚焦分析
用户只需特定模块（如仅需方法论批判），只执行相关模块

---

## 错误处理

| 情况 | 处理方式 |
|------|---------|
| 材料不完整 | 说明覆盖范围限制，建议补充材料 |
| 非政治科学著作 | 调整分析框架（如改为历史学/社会学视角），但注明局限 |
| 用户要求简短 | 提供精简版，但保持批判深度 |
| 检索失败 | 基于现有材料分析，注明文献对话部分受限 |
