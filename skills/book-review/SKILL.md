## 设定任务

- 将任务视为学术专著分析，而非内容摘要。
- 默认输出中文学术综述型长文；除非用户要求简版，否则优先保留批判深度。
- 首次出现关键术语时给出中英对照，如“内生性 endogeneity”“本体论 ontology”。
- 按分析维度组织，而非按章节顺序复述；仅在用户明确要求时才转为章节式梳理。
- 将批判、替代解释和改进方案视为必需项，而非附加项。

## 先选模式

| 场景 | 默认动作 |
|------|---------|
| 用户提供完整材料（PDF、全文、长摘录） | 直接进入五模块分析；只在关键信息缺失时追问 |
| 用户只给书名/DOI | 先确认元数据，再检索学术书评与文献脉络 |
| 用户只给章节或片段 | 明确覆盖范围受限，只对可见材料做判断 |
| 用户只要某一部分 | 只执行相关模块，避免完整铺开 |

默认采用**单次整合输出**。只有在用户要求分步推进、材料极长、或任务明显高风险时，才切换为多轮确认模式。

## 读取流程与模块

- 遇到非琐碎任务，先读取 `workflow.md`。
- 进入具体模块时，再按需读取对应文件，避免一次性加载全部细节。

**快速流程**：
1. 判定材料与任务模式
2. 提取核心论点与研究类型
3. 分模块深度分析
4. 构建文献对话网络
5. 整合输出并标明范围限制

## 子领域校准

| 子领域 | 额外强调 |
|------|---------|
| 比较政治 | 因果机制、案例选择、竞争性解释、区域文献 |
| 国际关系 | 理论传统定位、层次分析、历史时期与国际结构 |
| 政治理论 | 概念界定、规范前提、论证链条、竞争规范理论 |
| 公共行政 | 制度设计、实施机制、组织能力、政策外部效度 |

## 质量标准

- [ ] 按分析维度组织，而非章节顺序
- [ ] 每个核心模块都包含评析，而非纯描述
- [ ] 主动构建学术对话，而非只复述作者引文
- [ ] 每个主要批判都配有替代解释、理论修正或方法改进
- [ ] 理论模型已被分解为因果链条、逻辑前提或博弈结构
- [ ] 方法评估与研究类型匹配，不强行套用实证模板
- [ ] 在材料有限时明确说明推断边界
